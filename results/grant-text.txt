ABOUT COP-PILOT:
COP-PILOT is an Innovation Action funded under the EU HORIZON program focused on developing a Collaborative Open Platform (COP) framework that enables end-to-end orchestration of services across diverse industry sectors. Building upon the foundations of edge computing, advanced 5G connectivity, and decentralized processing, COP-PILOT aims to overcome fragmented infrastructure and isolated service domains by providing a unified platform for secure, automated, and intelligent management of resources and data.

At its core, COP-PILOT integrates novel AI-driven tools and automation solutions with robust security features to facilitate the onboarding and management of complex applications. The project prioritizes modularity, ease-of-use, and industry-standard interoperability, leveraging multi-tiered data processing, policy-driven optimization, and dynamic reasoning for service orchestration. Through hierarchical orchestration and resource slicing, COP-PILOT enables scalable, reliable, and SLA-preserving deployment of vertical services.

TECHNICAL REQUIREMENTS (COP-PILOT Platform):
Two Primary Workflows for Open Call Participants:

Workflow 1 - Onboarding New Services (for Service Providers):
- Publish application to git-based repository (container images + deployment manifests)
- CI pipeline auto-translates to standardized service blueprints
- Service ordered via LLM-based portal chat
- Application must be containerized and expose performance metrics

Workflow 2 - Integrating New Infrastructure Domains (for Infrastructure Owners):
- Register private domain via portal
- Auto-Pilot process establishes zero-trust encrypted connections via SIF
- Must run COP-PILOT Domain Orchestrator (DO) and Data Management (DM) components
- Use compatible infrastructure controllers (e.g., Kubernetes)

Key Platform Technologies:
- Business Portal: Next.js, React, Keycloak, LangChain, Ollama/vLLM
- E2E Service Orchestrator (ESO): Apache Quarkus, Prometheus, Grafana, Apache Kafka
- Domain Orchestrator (DO): Spring Boot, Keycloak, Flowable, Apache ActiveMQ, MCP
- Data Management: Orion/Scorpio/Stellio Context Broker, Eclipse Arrowhead
- Secure Integration Fabric (SIF): OpenZiti (Go), X.509/PKI, mTLS
- Compute Controller: Kubernetes, OpenShift, OCM, ColonyOS

Standards Compliance Required:
- TMForum (TMF) APIs for service/resource orchestration
- ETSI ZSM for zero-touch network management
- NGSI-LD / NGSI-v2 for data management
- FIWARE, Eclipse Arrowhead for IoT
- CNCF SPIFFE, OAuth 2.0, OpenID Connect for security
